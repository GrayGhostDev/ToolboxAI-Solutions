    ğŸš¦ Startup FlowÂ¶

    Phase 1: InfrastructureÂ¶

    1. PostgreSQL Database
    2. Starts first as core dependency
    3. Health check: pg_isready

    4. Creates multiple databases on init

    5. Redis Cache

    6. Starts alongside PostgreSQL
    7. Health check: redis-cli ping
    8. Configured with persistence

    Phase 2: Backend ServicesÂ¶

    1. FastAPI Backend
    2. Waits for PostgreSQL and Redis
    3. Health check: /health endpoint
    4. Runs database migrations on startup

    Phase 3: Frontend ServicesÂ¶

    1. Dashboard Frontend
    2. Waits for FastAPI to be healthy
    3. Health check: HTTP request to Vite server
    4. Environment variables injected at runtime

    Phase 4: Support ServicesÂ¶

    1. MCP Server, Agent Coordinator, Flask Bridge
    2. Start after core services are healthy
    3. Each has independent health checks

    ğŸ” TroubleshootingÂ¶

    Common Issues and SolutionsÂ¶

    1. Docker Daemon Not RunningÂ¶

    Error: Cannot connect to the Docker daemon Solution: 

    # Start Docker Desktop
open -a Docker

# Wait for Docker to be ready
docker version

    2. Container Name ConflictsÂ¶

    Error: The container name is already in use Solution: 

    # Stop and remove existing containers
docker-compose -f infrastructure/docker/docker-compose.dev.yml down
docker container prune

    3. Port Already in UseÂ¶

    Error: bind: address already in use Solution: 

    # Find process using port
lsof -i :5179
lsof -i :8009

# Kill process
kill -9 <PID>

    4. Dashboard Not LoadingÂ¶

    Issue: White screen or connection errors Solution: 

    # Check backend is running
curl http://localhost:8009/health

# Check environment variables
docker-compose -f infrastructure/docker/docker-compose.dev.yml exec dashboard-frontend env | grep VITE

# Rebuild with fresh dependencies
docker-compose -f infrastructure/docker/docker-compose.dev.yml build --no-cache dashboard-frontend

    5. Authentication ErrorsÂ¶

    Issue: ClerkAuthProvider errors Solution: 

    # Ensure legacy auth is enabled
export VITE_ENABLE_CLERK_AUTH=false
export ENABLE_CLERK_AUTH=false

# Restart services
docker-compose -f infrastructure/docker/docker-compose.dev.yml restart dashboard-frontend
