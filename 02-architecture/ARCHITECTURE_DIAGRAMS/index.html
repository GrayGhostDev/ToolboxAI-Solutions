    Data Flow Architecture¶

    Request/Response Flow¶

    sequenceDiagram
    actor User
    participant Dashboard
    participant FastAPI
    participant Service
    participant Database
    participant Pusher
    participant Agent

    User->>Dashboard: Interact with UI
    Dashboard->>FastAPI: HTTP POST /api/v1/content/generate
    FastAPI->>FastAPI: Validate JWT Token
    FastAPI->>Service: Call ContentGenerationService
    Service->>Database: Check existing content
    Database-->>Service: Return query results
    Service->>Agent: Trigger AI Agent
    Agent->>Agent: Generate content
    Agent-->>Pusher: Broadcast progress event
    Pusher-->>Dashboard: Real-time update
    Agent-->>Service: Return generated content
    Service->>Database: Save content
    Database-->>Service: Confirm save
    Service-->>FastAPI: Return response
    FastAPI-->>Dashboard: JSON response
    Dashboard-->>User: Display result

    Real-time Event Flow¶

    sequenceDiagram
    participant Backend
    participant Pusher Service
    participant Pusher Cloud
    participant Dashboard
    participant User

    Backend->>Pusher Service: trigger_event("content-progress", data)
    Pusher Service->>Pusher Cloud: Publish to channel
    Pusher Cloud->>Dashboard: WebSocket event delivery
    Dashboard->>Dashboard: usePusherEvent hook callback
    Dashboard->>User: Update UI with progress

    Note over Dashboard,Pusher Cloud: Auto-reconnect on disconnect
    Note over Backend,Pusher Service: Event history logging

    Authentication Flow¶

    JWT Authentication with Clerk¶

    sequenceDiagram
    actor User
    participant Dashboard
    participant Clerk
    participant FastAPI
    participant Database
    participant Protected Resource

    User->>Dashboard: Click "Sign In"
    Dashboard->>Clerk: Redirect to Clerk
    Clerk->>Clerk: User authenticates
    Clerk-->>Dashboard: Return JWT Token
    Dashboard->>Dashboard: Store token in state

    User->>Dashboard: Request protected resource
    Dashboard->>FastAPI: GET /api/v1/users/me<br/>Authorization: Bearer {token}
    FastAPI->>FastAPI: Validate JWT signature
    FastAPI->>FastAPI: Check token expiration
    FastAPI->>Database: Get user from token claims
    Database-->>FastAPI: User details
    FastAPI->>Protected Resource: Access granted
    Protected Resource-->>FastAPI: Resource data
    FastAPI-->>Dashboard: JSON response
    Dashboard-->>User: Display data

    Note over Dashboard,Clerk: OAuth 2.0 / OIDC
    Note over FastAPI: JWT validation middleware

    Role-Based Access Control (RBAC)¶

    flowchart TB
    Request[Incoming Request] --> ValidateToken{Valid JWT?}
    ValidateToken -->|No| Unauthorized[401 Unauthorized]
    ValidateToken -->|Yes| ExtractClaims[Extract Token Claims]

    ExtractClaims --> GetUser[Get User from Database]
    GetUser --> CheckRole{Check User Role}

    CheckRole -->|Admin| AdminAccess[Full Access]
    CheckRole -->|Teacher| TeacherAccess[Create/Edit Content]
    CheckRole -->|Student| StudentAccess[View Only]

    AdminAccess --> ExecuteAction[Execute Action]
    TeacherAccess --> CheckPermission{Has Permission?}
    StudentAccess --> CheckPermission

    CheckPermission -->|Yes| ExecuteAction
    CheckPermission -->|No| Forbidden[403 Forbidden]

    ExecuteAction --> Success[200 Success]

    style ValidateToken fill:#FFD700
    style CheckRole fill:#87CEEB
    style CheckPermission fill:#90EE90

    AI Agent System¶

    Agent Orchestration Flow¶

    graph TB
    Start[Content Generation Request] --> Orchestrator[Main Orchestrator]

    Orchestrator --> AnalyzeRequest[Analyze Request Type]

    AnalyzeRequest --> Decision{Content Type?}

    Decision -->|Lesson| ContentAgent[Content Agent<br/>LangChain LCEL]
    Decision -->|Quiz| QuizAgent[Quiz Agent<br/>LangChain LCEL]
    Decision -->|Script| ScriptAgent[Script Agent<br/>LangChain LCEL]
    Decision -->|Terrain| TerrainAgent[Terrain Agent<br/>LangChain LCEL]

    ContentAgent --> ReviewAgent[Review Agent<br/>Quality Check]
    QuizAgent --> ReviewAgent
    ScriptAgent --> ReviewAgent
    TerrainAgent --> ReviewAgent

    ReviewAgent --> TestAgent[Test Agent<br/>Validation]

    TestAgent --> Supervisor[Supervisor Agent<br/>Final Review]

    Supervisor --> Quality{Quality OK?}

    Quality -->|Yes| SaveResults[Save to Database]
    Quality -->|No| Retry[Retry with Feedback]

    Retry --> ContentAgent
    Retry --> QuizAgent
    Retry --> ScriptAgent
    Retry --> TerrainAgent

    SaveResults --> BroadcastEvent[Broadcast Pusher Event]

    BroadcastEvent --> End[Return to User]

    Agent Communication Protocol¶

    sequenceDiagram
    participant Client
    participant Orchestrator
    participant ContentAgent
    participant LangSmith
    participant OpenAI
    participant Database
    participant Pusher

    Client->>Orchestrator: Generate content request
    Orchestrator->>ContentAgent: Delegate task
    ContentAgent->>LangSmith: Log trace start
    ContentAgent->>OpenAI: Call LLM with prompt
    OpenAI-->>ContentAgent: Generated content
    ContentAgent->>LangSmith: Log trace end
    ContentAgent->>Database: Save intermediate results
    ContentAgent->>Pusher: Broadcast progress (25%)
    ContentAgent->>OpenAI: Refine content
    OpenAI-->>ContentAgent: Refined content
    ContentAgent->>Pusher: Broadcast progress (50%)
    ContentAgent->>Database: Save refined content
    ContentAgent->>Pusher: Broadcast progress (100%)
    ContentAgent-->>Orchestrator: Return final content
    Orchestrator-->>Client: Complete response

    Note over ContentAgent,LangSmith: Full observability in LangSmith dashboard
    Note over ContentAgent,Pusher: Real-time progress updates

    Performance Optimization¶

    Caching Strategy¶

    flowchart TB
    Request[Incoming Request] --> CheckCache{Check Redis Cache}

    CheckCache -->|Hit| ServeCache[Serve from Cache<br/>~10ms response]
    CheckCache -->|Miss| CheckDB{Check Database}

    CheckDB -->|Found| QueryDB[Query PostgreSQL<br/>~100ms response]
    CheckDB -->|Not Found| Generate[Generate with AI Agent<br/>~5-10s response]

    QueryDB --> UpdateCache1[Update Redis Cache<br/>TTL: 1 hour]
    Generate --> UpdateCache2[Update Redis Cache<br/>TTL: 24 hours]

    UpdateCache1 --> Response[Return Response]
    UpdateCache2 --> Response
    ServeCache --> Response

    style CheckCache fill:#FFD700
    style ServeCache fill:#90EE90
    style Generate fill:#FF6B6B
    style Response fill:#87CEEB
