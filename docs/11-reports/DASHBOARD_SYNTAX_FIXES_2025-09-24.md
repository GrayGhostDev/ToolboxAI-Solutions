### 1. Fixed Regular Expression Patterns in `robloxSpecExtractor.ts`

#### Issue
Multiple regular expression patterns had broken character classes `[^` that were split across lines, causing "Unterminated regular expression literal" errors.

#### Files Modified
- `apps/dashboard/src/utils/robloxSpecExtractor.ts`

#### Specific Changes

**Lines 136-138: Learning Objectives Extraction**
```typescript
// Before (broken):
/(?:objective|learning objective|goal)s?\s*:?\s*([^
]+)/i,
/(?:teach|learn|practice|study)\s+([^
.!?]+)/i,
/(?:about|covering|focusing on)\s+([^

// After (fixed):
/(?:objective|learning objective|goal)s?\s*:?\s*([^\n]+)/i,
/(?:teach|learn|practice|study)\s+([^\n.!?]+)/i,
/(?:about|covering|focusing on)\s+([^\n.!?]+)/i
```

**Lines 81-82: Theme Extraction**
```typescript
// Before (broken):
/(?:theme|style)\s*:?\s*([^
.!?]{3,50})/i,
/(?:set in|takes place in|located in)\s+([^
.!?]{3,50})/i

// After (fixed):
/(?:theme|style)\s*:?\s*([^\n.!?]{3,50})/i,
/(?:set in|takes place in|located in)\s+([^\n.!?]{3,50})/i
```

#### Root Cause
The regex patterns were attempting to match "not newline" characters using `[^\n]` but the newline character itself was literally inserted in the code, breaking the regex syntax. The character class `[^` was left unterminated.

## Technical Details

### Regular Expression Fix Explanation
The pattern `[^...]` in regex means "match any character except those in the brackets". When writing `[^\n]`, we're saying "match any character except newline". The broken patterns had actual newlines inserted between `[^` and the closing `]`, which made the regex parser unable to find the closing bracket.

- **Before Fix:** TypeScript compilation failed completely with syntax errors
