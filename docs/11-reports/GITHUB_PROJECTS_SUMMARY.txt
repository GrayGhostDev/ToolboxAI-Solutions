RECOMMENDED CUSTOM FIELDS
═══════════════════════════════════════════════════════════════════════════

Configure these fields in GitHub UI after creating projects:

PRIORITY (Single Select):
  • Critical  - Immediate action required
  • High      - Important, schedule soon
  • Medium    - Normal priority
  • Low       - Nice to have

STATUS (Single Select):
  • Backlog      - Not yet planned
  • Todo         - Planned for current sprint
  • In Progress  - Active development
  • In Review    - Under review
  • Done         - Completed
  • Cancelled    - Not proceeding

SIZE (Single Select):
  • XS  - < 1 day
  • S   - 1-2 days
  • M   - 3-5 days
  • L   - 1-2 weeks
  • XL  - > 2 weeks

SPRINT (Text):
  • Current sprint identifier (e.g., "Sprint Q1 2025", "Sprint 23")

DUE DATE (Date):
  • Target completion date
