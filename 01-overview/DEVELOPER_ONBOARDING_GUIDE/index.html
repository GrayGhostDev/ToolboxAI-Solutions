    ğŸš€ Quick Start (5 minutes)Â¶

    Option 1: Docker Development (Recommended)Â¶

    # 1. Clone the repository
git clone https://github.com/ToolBoxAI-Solutions/toolboxai.git
cd toolboxai

# 2. Copy environment template
cp .env.example .env

# 3. Generate secure secrets (macOS/Linux)
echo "JWT_SECRET_KEY=$(openssl rand -hex 32)" >> .env
echo "POSTGRES_PASSWORD=$(openssl rand -base64 32)" >> .env
echo "REDIS_PASSWORD=$(openssl rand -base64 24)" >> .env

# 4. Start all services
docker compose -f infrastructure/docker/compose/docker-compose.yml \
               -f infrastructure/docker/compose/docker-compose.dev.yml up -d

# 5. Verify services are running
docker compose ps

# 6. Access the platform
open http://localhost:5179  # Dashboard
open http://localhost:8009/docs  # API Documentation

    That's it! You now have a fully functional development environment.

    Option 2: Native DevelopmentÂ¶

    For local development without Docker (useful for debugging):

    # 1. Clone the repository
git clone https://github.com/ToolBoxAI-Solutions/toolboxai.git
cd toolboxai

# 2. Setup Python virtual environment
python3 -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# 3. Install Python dependencies
pip install -r requirements.txt

# 4. Setup Node.js dependencies
cd apps/dashboard
npm install  # Use --no-bin-links if on external drive
cd ../..

# 5. Setup databases (requires PostgreSQL and Redis running)
# PostgreSQL: brew install postgresql (macOS)
# Redis: brew install redis (macOS)

# 6. Run backend
cd apps/backend
uvicorn main:app --host 127.0.0.1 --port 8009 --reload

# 7. Run frontend (in another terminal)
cd apps/dashboard
npm run dev

    Technology StackÂ¶

    BackendÂ¶

    • Framework: FastAPI 0.116.1 (async Python web framework)
    • AI/ML: LangChain 0.3.26+, LangGraph 0.2.65+
    • Database: PostgreSQL 16 (primary), Supabase (optional)
    • Cache: Redis 7
    • Real-time: Pusher Channels (primary solution)
    • Auth: JWT + optional Clerk integration
    • Validation: Pydantic v2 (2.9.0+)

    FrontendÂ¶

    • Framework: React 19.1.0 with concurrent features
    • Language: TypeScript 5.9.2 (strict mode)
    • Build Tool: Vite 6.0.1
    • UI Library: Mantine 8.3.1
    • Icons: Tabler Icons React 3.35.0
    • State Management: Redux Toolkit 2.2.7
    • Real-time: Pusher-js 8.4.0
    • Testing: Vitest 3.2.4 + React Testing Library

    3. Test Your ChangesÂ¶

# Frontend tests

npm run test:coverage       # With coverage report

    4. Commit Your WorkÂ¶

    # Stage your changes

- Include tests for profile operations

# fix: bug fix
# docs: documentation changes
# test: test additions/changes
# refactor: code refactoring

    Writing TestsÂ¶

    Backend Tests (Python)Â¶

    # tests/unit/test_user_profile.py
import pytest
from fastapi.testclient import TestClient
from apps.backend.main import app

client = TestClient(app)

def test_get_user_profile():
    """Test retrieving user profile."""
    response = client.get(
        "/api/v1/users/me/profile",
        headers={"Authorization": "Bearer test-token"}
    )
    assert response.status_code == 200
    data = response.json()
    assert "user_id" in data
    assert "email" in data

@pytest.mark.integration
def test_update_profile():
    """Test updating user profile."""
    payload = {"display_name": "John Doe"}
    response = client.patch(
        "/api/v1/users/me/profile",
        json=payload,
        headers={"Authorization": "Bearer test-token"}
    )
    assert response.status_code == 200

    Python (Backend)Â¶

    # Use type hints everywhere
from typing import Optional, List

def get_user(user_id: int) -> Optional[User]:
    """Retrieve user by ID.

    Args:
        user_id: The unique user identifier

    Returns:
        User object if found, None otherwise
    """
    return db.query(User).filter(User.id == user_id).first()

# Use Pydantic v2 for validation
from pydantic import BaseModel, Field, field_validator

class UserProfile(BaseModel):
    """User profile data model."""
    user_id: int = Field(..., description="Unique user identifier")
    email: str = Field(..., description="User email address")
    display_name: Optional[str] = None

    @field_validator('email')
    @classmethod
    def validate_email(cls, v: str) -> str:
        """Validate email format."""
        if '@' not in v:
            raise ValueError('Invalid email')
        return v.lower()
